# 日志规范

## 错误日志打印

【推荐】打印错误日志时，将错误栈（Throwable）也打印出来。

## 正确示例

```java
try {
    // 业务逻辑
} catch (Exception e) {
    log.error("操作失败", e); // 打印完整错误栈
    throw e;
}
```

## 错误示例

```java
try {
    // 业务逻辑
} catch (Exception e) {
    log.error("操作失败: " + e.getMessage()); // 只打印消息，不打印错误栈
    throw e;
}
```

## 为什么需要打印完整错误栈

1. **快速定位问题**：完整错误栈可以显示异常的完整调用链，帮助快速定位问题根源
2. **减少排查时间**：只有错误消息时，很难判断异常发生的位置和原因
3. **保留根因信息**：有时候根异常在最内层，只打印Message会丢失重要信息

## 日志级别使用建议

| 场景 | 日志级别 | 说明 |
|------|---------|------|
| 正常业务流程 | DEBUG | 详细调试信息 |
| 重要步骤完成 | INFO | 一般信息 |
| 可能的问题 | WARN | 警告信息，但不影响流程 |
| 错误和异常 | ERROR | 错误信息，需要关注 |
