# MySQL 开发规范

本技能提供 MySQL 数据库全栈开发规范，适用于：业务表设计、索引优化、SQL 编写、ORM 映射、代码审查和数据库评审。

## 规范等级说明

| 标记 | 含义 | 约束力 |
|------|------|--------|
| 【强制】 | 必须遵守，违反将导致评审不通过 | 最高 |
| 【推荐】 | 强烈建议遵守，提升质量和性能 | 高 |
| 【参考】 | 可选优化项，视情况采纳 | 中 |

## 按需加载 Reference

**根据当前任务加载对应 reference 文件，绝不要一次性加载所有文件：**

| 任务类型 | 加载文件 |
|----------|----------|
| 建表、字段命名、字段类型选择 | `references/table-design.md` |
| 索引创建、索引优化、查询性能 | `references/index-design.md` |
| SQL 编写、查询优化、DML 操作 | `references/sql-writing.md` |
| MyBatis 映射、POJO 设计、XML 配置 | `references/orm-mapping.md` |

**多任务时**：按顺序分别读取多个 reference 文件，每次只读当前需要的。

## 通用检查清单

当用户提出以下请求时，必须激活此技能：

- "帮我看看这个表设计合不合理"
- "帮我设计一个 xxx 表"
- "这个 SQL 怎么优化"
- "帮我评审这段 MyBatis 代码"
- "创建表的时候要注意什么"
- "这个字段用什么类型"
- "索引怎么建"
- 任何涉及 MySQL 建表、字段、索引、SQL 的问题

## 使用流程

1. 识别用户任务类型（建表/索引/SQL/ORM）
2. 加载对应 reference 文件
3. 应用规范检查，如有必要提出修改建议
4. 对【强制】项必须指出问题，对【推荐】项说明理由
