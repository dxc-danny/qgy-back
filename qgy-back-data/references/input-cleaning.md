# 输入清洗规范

## 清洗工具类

使用 `qinggouyun-common:QgyStrUtil` 工具类进行字符串清洗：

| 方法 | 用途 |
|------|------|
| cleanBlank | 去掉首尾及中间的特殊字符 |
| cleanBlankKeepVisible | 去掉首尾及中间的特殊字符，但保留可见空格 |

## 主体名称清洗

【强制】以下主体名称类型必须在执行业务逻辑或持久化到存储中间件之前进行清洗：
- 姓名
- 商户名称
- 客户名称
- 公司名称
- 点位名称
- 店铺名称
- 活动名称
- 广告名称
- 商品名称

**清洗方式**：调用 `QgyStrUtil.cleanBlank` 去掉首尾及中间的特殊字符

**特殊情况**：如果某某名称中间需要保留空格，调用 `QgyStrUtil.cleanBlankKeepVisible`

## 主体编码清洗

【强制】以下主体编码类型必须在执行业务逻辑或持久化到存储中间件之前进行清洗：
- 商品条码
- 商品编码
- 点位编码
- 组织编码
- 商户商品条码

**清洗方式**：调用 `QgyStrUtil.cleanBlank` 去掉首尾及中间的特殊字符

## 其它信息清洗

【强制】以下其它信息类型必须在执行业务逻辑或持久化到存储中间件之前进行清洗：
- 提货卷码
- 手机号码
- 电话号码

**清洗方式**：调用 `QgyStrUtil.cleanBlank` 去掉首尾及中间的特殊字符

## 清洗时机

清洗必须在以下时机之前执行：
1. 执行业务逻辑之前
2. 持久化到MySQL之前
3. 持久化到ElasticSearch之前
4. 持久化到MongoDB之前
5. 持久化到Redis之前

## 不确认时

如果不确认是否需要清洗或如何清洗，需要找技术主管沟通。
