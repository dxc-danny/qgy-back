# 接口设计规范

## 接口变更处理

当service接口需要发生变更时（主要针对common模块），必须选择以下方案之一：

### 方案一：全部更改
将所有相关调用全部更改干净（包括其他子项目），如不清楚其他子项目可联系上级领导之后再进行操作。

### 方案二：废弃旧接口
1. 将现有接口做作废标记@Deprecated并说明原因和推荐使用的新接口
2. 写新的接口，并实现
3. 向上级领导汇报

## 返回数据量控制

【强制】接口只返回大前端必需的字段，返回的数据量越小越好：

| 接口类型 | 数据量限制 | 说明 |
|---------|-----------|------|
| 高频接口 | 1K以内 | 频繁调用的接口需要严格控制 |
| 低频接口 | 128K以内 | 不常调用的接口可以适当放宽 |
| 特殊情况 | 跟技术经理沟通确认 | 需要提前沟通 |

## 返回字段原则

1. 只返回前端必需的字段
2. 避免返回冗余字段增加网络传输
3. 大字段（如描述、详情等）需要评估是否必需
