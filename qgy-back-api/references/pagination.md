# 分页限制规范

## 分页配置表

| 场景 | 最大页大小 | 推荐值 | 申请要求 |
|------|-----------|-------|---------|
| 前端接口 | 100 | 10 | >100需向技术主管申请 |
| 普通深分页 | 100 | - | >100需向技术主管申请 |
| 导出深分页 | 500 | - | >500需向技术主管申请 |

## 前端接口分页

【强制】前端接口原则上页大小禁止大于100，推荐为10，大于100需要向技术主管申请

## 普通深分页

【强制】普通场景深分页【必须解决深页面问题】接口原则上页大小禁止大于100，大于100需要向技术主管申请

## 导出深分页

【强制】导出数据数据迁移场景深分页【必须解决深页面问题】接口原则上页大小禁止大于500，大于500需要向技术主管申请

## 深分页问题解决方案

解决深分页问题的常用方案：

1. **游标分页（Cursor Pagination）**：使用id或时间戳作为游标，避免OFFSET
2. **SearchAfter**：ES中使用search_after基于唯一字段进行深度分页
3. **滚动查询（Scroll）**：ES中使用scroll API进行大规模数据导出
4. **分段查询**：按ID范围或时间范围分段查询
